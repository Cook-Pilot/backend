# 03. CookPilot MVP 설계

작성 기준: 2026-07-10 확정 MVP

## 1. MVP 한 문장

CookPilot MVP는 조리 화면에서만 STT/TTS 음성 조리 모드를 활성화하고, 레시피 단계와 타이머는 로컬에서 자동 진행하며, 사용자의 예외 상황 발화만 LLM이 현재 단계 맥락으로 피드백하는 앱 데모다. 조리 후 피드백은 개인 레시피 버전으로 저장되어 다음 조리 때 개선된 레시피로 제공된다.

## 2. MVP 핵심 판단

| 항목 | 결정 |
|---|---|
| 핵심 경험 | 조리 화면 한정 음성 조리 모드 |
| 화면 역할 | 현재 단계와 타이머를 계속 보여주는 상태판 |
| 음성 역할 | 손이 어려운 상황에서 명령과 예외 질문 처리 |
| STT/TTS | MVP 필수 포함 |
| LLM 사용 | 로컬 처리 불가능한 예외 상황과 개인화 반영에만 사용 |
| 비용 전략 | 조리 시간 전체가 아니라 판단 발화에만 LLM 비용 발생 |
| 제외 | 백그라운드 음성 호출, 커스텀 호출어, 사진 판정 |

## 3. 핵심 루프

```text
레시피 선택
-> 조리 시작
-> 현재 단계 표시 + TTS 안내
-> 단계별 타이머 자동 진행
-> 조리 중 STT 음성 명령 수신
-> 단순 명령은 로컬 처리
-> 예외 상황은 LLM 피드백
-> 조리 후 피드백 입력
-> 개인 레시피 버전 업데이트
-> 다음 조리 때 개인화 버전 제공
```

## 4. MVP 포함 범위

- 레시피 선택
- 조리 시작
- 현재 단계 표시
- TTS 단계 안내
- 단계별 타이머 자동 진행
- 조리 중 STT 음성 명령
- 단순 명령 로컬 처리
- 예외 상황 LLM 피드백
- 버튼 fallback
- 세션 이벤트 기록
- 조리 후 피드백
- 개인 레시피 버전 업데이트
- 다음 조리 개인화 버전 제공

## 5. MVP 제외 범위

- 인기 레시피
- 추천 알고리즘 고도화
- 남의 버전 추천
- 사진 기반 익힘/농도 판정
- 커뮤니티
- 장보기/냉장고 연동
- 백그라운드 음성 호출
- 커스텀 호출어
- 대규모 레시피 DB

## 6. Active Cooking Voice Mode

```text
앱 실행
-> 레시피 선택
-> 조리 시작
-> 조리 화면 진입
-> 음성 조리 모드 ON
-> 조리 중 음성 명령 수신
-> 조리 완료/중단/화면 이탈
-> 음성 조리 모드 OFF
```

| 항목 | 지원 여부 |
|---|---|
| 조리 화면 포그라운드 음성 수신 | 지원 |
| 조리 세션 중 음성 명령 | 지원 |
| 화면 이탈 시 음성 수신 중단 | 지원 |
| 백그라운드 음성 수신 | 미지원 |
| 잠금화면 음성 호출 | 미지원 |
| 커스텀 호출어 | 미지원 |

## 7. 라면 데모 시나리오

```text
1. 사용자가 라면 레시피를 선택한다.
2. 앱이 말한다: "물 500ml를 넣고 3분간 끓이세요. 타이머를 시작합니다."
3. 유저가 말하지 않으면 3분 타이머가 자동 진행된다.
4. 3분 후 앱이 말한다: "건더기, 분말스프, 면을 넣고 3분간 끓이세요."
5. 다음 3분 타이머가 자동 시작된다.
6. 사용자가 말한다: "어, 지금 물이 안 끓어."
7. 앱은 현재 단계와 남은 시간을 LLM에 전달한다.
8. 앱이 말한다: "아직 끓지 않으면 1분 더 끓이고, 기포가 올라오면 다음 단계로 넘어가세요."
9. 조리 완료 후 사용자가 "너무 짰다"라고 피드백한다.
10. 다음 조리 때 분말스프 감소 또는 물 조정이 반영된 내 버전을 제안한다.
```

## 8. 로컬 명령 세트

| 의도 | 예시 | 동작 |
|---|---|---|
| 다음 단계 | "다음", "다음 단계" | 단계 index 증가 |
| 이전 단계 | "이전", "전 단계" | 단계 index 감소 |
| 반복 | "다시 말해줘" | 현재 단계 TTS 재생 |
| 타이머 연장 | "1분 더" | 로컬 타이머 연장 |
| 타이머 제어 | "멈춰", "다시 시작" | 타이머 pause/resume |
| 단계 확인 | "지금 뭐 해야 해?" | 현재 단계 안내 |

## 9. LLM 처리 세트

| 유형 | 예시 | 응답 기준 |
|---|---|---|
| 상태 판단 | "물이 안 끓어" | 현재 단계 기준으로 대기/연장 안내 |
| 익힘 판단 | "면이 덜 익었어" | 추가 시간과 확인 기준 제시 |
| 간 조절 | "너무 짜" | 희석/재료 추가/다음 조리 조정 안내 |
| 재료 문제 | "파가 없어" | 생략/대체 가능 여부 안내 |
| 안전 질문 | "상한 것 같아" | 단정 금지, 폐기/확인 기준 우선 |

## 10. 데모 성공 기준

| 항목 | 기준 |
|---|---|
| 단계 진행 | TTS 안내와 타이머 자동 시작이 동작한다. |
| STT | 조리 중 주요 발화를 텍스트로 받는다. |
| 로컬 처리 | 다음/반복/1분 더가 LLM 없이 처리된다. |
| LLM 피드백 | "물이 안 끓어"에 현재 단계 기준으로 답한다. |
| 조리 후 피드백 | "너무 짰다" 같은 코멘트를 저장한다. |
| 개인화 | 다음 조리 때 수정된 내 버전이 보인다. |
