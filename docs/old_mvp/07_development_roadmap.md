# 07. CookPilot 개발 로드맵

작성 기준: 2026-07-10 확정 MVP  
목표 일정: 2026-07-26 동작 가능한 앱 데모

## 1. 데모 목표

2026-07-26까지 보여줘야 하는 핵심 시나리오:

```text
1. 사용자가 레시피를 선택한다.
2. 조리 시작을 누른다.
3. 앱이 현재 단계를 화면에 표시하고 TTS로 안내한다.
4. 타이머가 자동으로 시작된다.
5. 사용자가 "다음", "다시 말해줘", "1분 더"를 말하면 로컬 처리된다.
6. 사용자가 "물이 안 끓어" 같은 예외 상황을 말하면 LLM이 현재 단계 기준으로 답한다.
7. 조리 완료 후 사용자가 "너무 짰다" 같은 피드백을 남긴다.
8. 앱이 개인 레시피 버전을 업데이트한다.
9. 같은 레시피를 다시 열면 개선된 내 버전이 제안된다.
```

## 2. 개발 우선순위

| 순위 | 영역 | 이유 |
|---|---|---|
| 1 | Recipe Runner | 모든 흐름의 기반 |
| 2 | 로컬 타이머 | 조리 진행의 신뢰성 |
| 3 | 조리 화면 UI | 핵심 사용자 경험 |
| 4 | TTS/STT | CookPilot 차별점 |
| 5 | Command Router | 비용과 안정성 제어 |
| 6 | LLM 예외 피드백 | 실시간 코치 가치 |
| 7 | 개인 레시피 버전 | 락인과 반복 개인화 |

## 3. Sprint 1. Recipe Runner

### 목표

음성/LLM 없이도 레시피를 단계별로 진행할 수 있는 기반을 만든다.

### 작업

- MVP 레시피 seed 작성
- 레시피 단계/타이머 데이터 구조 정의
- 조리 세션 생성
- 현재 단계 표시
- 다음/이전/반복
- 로컬 타이머 자동 시작
- 타이머 종료 후 다음 단계 안내 트리거

### 완료 기준

- 사용자가 버튼만으로 레시피를 처음부터 끝까지 진행할 수 있다.
- 단계별 타이머가 자동 시작되고 종료된다.

## 4. Sprint 2. STT/TTS 조리 모드

### 목표

조리 화면에서만 음성 입출력이 동작하게 한다.

### 작업

- 조리 화면 진입 시 음성 모드 시작
- 화면 이탈/완료/중단 시 음성 모드 종료
- TTS 단계 안내
- STT 결과 수신
- 음성 상태 UI
- 버튼 fallback 유지

### 완료 기준

- 단계 시작 시 TTS가 안내를 읽는다.
- 조리 중 사용자의 짧은 발화를 텍스트로 받을 수 있다.
- 조리 화면을 벗어나면 음성 수신이 종료된다.

## 5. Sprint 3. Command Router + LLM

### 목표

단순 명령은 로컬에서 처리하고, 예외 상황만 LLM으로 보낸다.

### 작업

- 로컬 명령 매칭
- LLM 호출 조건 정의
- 현재 레시피/단계/타이머 payload 구성
- LLM 응답 구조 정의
- LLM 응답 TTS 출력
- 세션 이벤트 저장

### 완료 기준

- "다음", "다시 말해줘", "1분 더"는 LLM 없이 처리된다.
- "물이 안 끓어"는 LLM이 현재 단계 기준으로 답한다.

## 6. Sprint 4. Personal Recipe Loop

### 목표

조리 후 피드백이 다음 조리 버전으로 반영되는 루프를 만든다.

### 작업

- 조리 후 피드백 화면
- 만족도/코멘트 저장
- 피드백 구조화
- 개인 레시피 버전 생성
- 같은 레시피 재진입 시 개인 버전 로드

### 완료 기준

- "너무 짰다" 같은 피드백이 개인 레시피 수정안으로 저장된다.
- 다음 조리 때 수정된 내 버전이 보인다.

## 7. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| STT 인식률 부족 | 버튼 fallback과 짧은 명령어 중심 설계 |
| TTS 품질/플랫폼 차이 | 우선 기기 내장 TTS 사용 |
| LLM 비용 증가 | 로컬 명령 우선, 예외 상황만 호출 |
| 조리 중 소음 | 음성 상태 UI와 재시도 안내 |
| 일정 부족 | 레시피 수를 줄이고 라면 등 1개 메뉴 데모를 우선 완성 |
| 백그라운드 제약 | MVP에서 조리 화면 포그라운드만 지원 |
