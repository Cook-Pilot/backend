# 01. CookPilot 요구사항 정리

작성 기준: 2026-07-10 확정 MVP  
목표 일정: 2026-07-26까지 동작 가능한 앱 데모

## 1. 제품 한 줄 정의

CookPilot은 조리 화면에서만 동작하는 STT/TTS 기반 음성 조리 모드로 레시피 단계를 진행하고, 단순 명령은 로컬에서 처리하며, 예외 상황은 LLM이 현재 단계 맥락으로 피드백하는 실시간 AI 조리 코치 앱이다.

조리 후 피드백은 개인 레시피 버전에 반영되어, 같은 레시피를 다시 조리할 때 더 나은 "내 버전"으로 제공된다.

## 2. 문제 정의

요리 초보자는 레시피를 알고 있어도 실제 조리 중 다음 문제를 겪는다.

- 손에 물, 기름, 재료가 묻어 화면 조작이 어렵다.
- 레시피를 보면서도 지금 다음 단계로 넘어가도 되는지 판단하기 어렵다.
- 물이 안 끓음, 간이 짬, 재료가 없음, 덜 익음 같은 예외 상황에서 막힌다.
- 한 번 실패하거나 수정한 경험이 다음 조리에 자동 반영되지 않는다.
- 기존 레시피 서비스는 조리 전 정보 제공에 강하지만, 조리 중 실시간 피드백과 조리 후 개인화 누적은 약하다.

## 3. 타깃 사용자

| 사용자 | 특징 | 주요 니즈 |
|---|---|---|
| 자취생 | 혼자 간단한 요리를 자주 시도 | 실패를 줄이고 빠르게 완성하고 싶음 |
| 사회초년생 | 요리 경험이 적고 시간 여유가 적음 | 손쉽게 따라갈 수 있는 안내 필요 |
| 신혼/1~2인 가구 | 인분과 취향 조정이 자주 필요 | 내 입맛에 맞는 반복 조리 버전 필요 |
| 요리 초보자 | 상태 판단과 타이밍에 약함 | 조리 중 바로 물어볼 수 있는 코치 필요 |

## 4. 핵심 사용자 가치

```text
조리 중 음성 기반 실시간 피드백
-> 조리 후 결과 피드백
-> 내 버전 레시피 수정
-> 다음 조리 때 개선된 레시피 제공
-> 반복할수록 개인화 강화
-> 개인 레시피 자산 축적과 락인
```

## 5. MVP 검증 질문

| 질문 | 검증 방법 | 성공 기준 |
|---|---|---|
| 조리 화면 한정 음성 모드가 실제로 쓸 만한가 | 실제 조리 시나리오 테스트 | 주요 단계 진행을 손 조작 없이 수행 |
| 단순 명령을 로컬에서 안정적으로 처리할 수 있는가 | 발화 로그 분석 | 다음/이전/반복/타이머 명령 대부분 성공 |
| 예외 상황 LLM 피드백이 조리 불안을 줄이는가 | 파일럿 인터뷰 | 사용자가 막힌 상황에서 다음 행동을 이해 |
| 조리 후 피드백이 다음 조리 개선으로 이어지는가 | 동일 레시피 2회 이상 테스트 | 2회차에 개인 버전 적용 |
| 개인 레시피가 재사용 이유가 되는가 | 재조리 의향 확인 | 사용자가 내 버전을 다시 쓰고 싶다고 응답 |

## 6. 기능 요구사항

| ID | 요구사항 | 설명 | 우선순위 |
|---|---|---|---|
| FR-01 | 레시피 선택 | MVP용 제한된 레시피 중 하나를 선택한다. | P0 |
| FR-02 | 조리 세션 생성 | 레시피 선택 후 조리 시작 시 active cooking session을 만든다. | P0 |
| FR-03 | 현재 단계 표시 | 조리 화면에 현재 조리법, 남은 시간, 주의사항을 보여준다. | P0 |
| FR-04 | TTS 단계 안내 | 단계 시작 시 현재 조리법을 음성으로 읽어준다. | P0 |
| FR-05 | 로컬 타이머 | 단계별 시간이 있으면 자동으로 타이머를 시작한다. | P0 |
| FR-06 | STT 음성 명령 | 조리 화면에서 사용자 발화를 텍스트로 변환한다. | P0 |
| FR-07 | 로컬 명령 처리 | 다음, 이전, 반복, 타이머 연장 등은 LLM 없이 처리한다. | P0 |
| FR-08 | LLM 예외 피드백 | 물이 안 끓음, 너무 짬, 덜 익음, 재료 없음 등은 LLM이 응답한다. | P0 |
| FR-09 | 버튼 fallback | 음성 실패 시 동일 조작을 버튼으로 수행한다. | P0 |
| FR-10 | 세션 이벤트 기록 | 단계 이동, 타이머 변경, 예외 상황, 피드백을 저장한다. | P0 |
| FR-11 | 조리 후 피드백 | 만족도와 자유 코멘트를 입력한다. | P0 |
| FR-12 | 개인 레시피 버전 | 피드백을 바탕으로 다음 조리용 내 버전을 생성한다. | P0 |
| FR-13 | 개인 버전 재사용 | 같은 레시피 재진입 시 개인화 버전을 제안한다. | P0 |

## 7. 비기능 요구사항

| ID | 요구사항 | 기준 |
|---|---|---|
| NFR-01 | 모바일 앱 우선 | 앱 중심 경험으로 설계한다. 웹은 핵심 우선순위가 아니다. |
| NFR-02 | 조리 화면 한정 음성 수신 | 조리 세션 중 조리 화면에서만 마이크를 활성화한다. |
| NFR-03 | 개인정보 | 백그라운드 상시 리스닝과 커스텀 호출어는 지원하지 않는다. |
| NFR-04 | 비용 제어 | 로컬 처리 가능한 명령은 LLM을 호출하지 않는다. |
| NFR-05 | 타이머 신뢰성 | LLM/API 지연과 무관하게 타이머는 로컬에서 동작한다. |
| NFR-06 | 응답 간결성 | 조리 중 LLM 응답은 1~2문장과 하나의 행동 중심으로 제한한다. |
| NFR-07 | 안전성 | 변질, 알레르기, 덜 익은 육류 등은 보수적으로 안내한다. |

## 8. MVP 제외 범위

- 인기 레시피
- 추천 알고리즘 고도화
- 남의 버전 추천
- 사진 기반 익힘/농도 판정
- 커뮤니티
- 장보기/냉장고 연동
- 백그라운드 음성 호출
- 커스텀 호출어
- 대규모 레시피 DB
