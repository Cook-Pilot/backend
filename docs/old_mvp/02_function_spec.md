# 02. CookPilot 기능 명세서

작성 기준: 2026-07-10 확정 MVP
갱신: 2026-07-21 — 조리 세션은 서버에 저장하지 않는다. 세션 진행(단계/타이머/이벤트)은
프론트가 로컬에서 관리하고, 서버에는 조리 결과(`post_cook_reviews`)와 개인 버전만 남는다.
아래 "핵심 데이터"의 `(프론트 로컬)` 표기는 서버 테이블이 없다는 뜻이다.

## 1. 기능 목록

| 기능 ID | 기능명 | 우선순위 | 관련 화면 | 핵심 데이터 |
|---|---|---|---|---|
| F-01 | 레시피 선택 | P0 | S-01 | `recipes`, `personal_recipe_versions` |
| F-02 | 조리 세션 생성 | P0 | S-02 | 세션 상태 (프론트 로컬) |
| F-03 | Recipe Runner | P0 | S-03 | `recipe_steps` + 진행 상태 (프론트 로컬) |
| F-04 | 로컬 타이머 | P0 | S-03 | 타이머/이벤트 (프론트 로컬) |
| F-05 | TTS 단계 안내 | P0 | S-03 | `tts_events` |
| F-06 | STT 음성 명령 | P0 | S-03 | `voice_transcripts` |
| F-07 | Command Router | P0 | S-03 | `command_events` |
| F-08 | LLM 예외 피드백 | P0 | S-03 | `ai_interactions` |
| F-09 | 버튼 fallback | P0 | S-03 | 이벤트 (프론트 로컬) |
| F-10 | 조리 후 피드백 | P0 | S-04 | `post_cook_reviews` |
| F-11 | 개인 레시피 버전 생성 | P0 | S-05 | `personal_recipe_versions` |
| F-12 | 다음 조리 개인화 | P0 | S-01, S-02 | `personal_recipe_versions` |

## F-01. 레시피 선택

| 항목 | 명세 |
|---|---|
| 목적 | 사용자가 MVP용 레시피 중 조리할 메뉴를 선택한다. |
| 입력 | 레시피 선택 |
| 처리 | 기본 레시피와 개인 버전 존재 여부를 조회한다. |
| 출력 | 조리 시작 화면 또는 조리 화면 |
| 예외 | 개인 버전이 없으면 기본 레시피로 시작한다. |
| 수락 기준 | 사용자는 최소 1개 이상의 MVP 메뉴를 선택해 조리 세션을 시작할 수 있다. |

## F-02. 조리 세션 생성

| 항목 | 명세 |
|---|---|
| 목적 | 이번 조리의 상태와 이벤트를 담을 세션을 프론트 로컬에 만든다(서버 저장 없음). |
| 입력 | 레시피 상세(단계/재료), 선택한 개인 버전 |
| 처리 | 조리 시작 시점의 단계와 개인화 조정값을 프론트가 스냅샷으로 고정한다. |
| 출력 | 프론트 로컬 세션 상태 (서버 API 없음) |
| 예외 | 앱 종료/새로고침 시 로컬 세션 복구는 프론트 책임 |
| 수락 기준 | 조리 중 단계/타이머/음성 이벤트가 서버 왕복 없이 로컬에서 처리된다. |

## F-03. Recipe Runner

| 항목 | 명세 |
|---|---|
| 목적 | 레시피 단계를 화면에 표시하고 현재 진행 상태를 관리한다. |
| 상태 | `READY`, `COOKING`, `PAUSED`, `REVIEW`, `COMPLETED`, `ABORTED` |
| 입력 | 다음, 이전, 반복, 타이머 완료, 조리 완료 |
| 처리 | 현재 단계 index와 단계별 instruction, timer를 갱신한다. |
| 출력 | 현재 단계, 남은 시간, 다음 행동 |
| 수락 기준 | LLM/API가 없어도 단계 이동과 타이머 진행은 가능해야 한다. |

## F-04. 로컬 타이머

| 항목 | 명세 |
|---|---|
| 목적 | 조리 시간 관리를 AI와 독립적으로 수행한다. |
| 입력 | 자동 시작, 연장, 정지, 재개, 완료 |
| 처리 | 클라이언트 기준으로 카운트다운하며 이벤트를 저장한다. |
| 출력 | 남은 시간, 완료 이벤트, 다음 단계 안내 트리거 |
| 수락 기준 | 단계 시작 시 지정된 시간이 있으면 타이머가 자동 시작된다. |

## F-05. TTS 단계 안내

| 항목 | 명세 |
|---|---|
| 목적 | 손 조작 없이 현재 조리 행동을 들을 수 있게 한다. |
| 입력 | 현재 단계 instruction, LLM feedback |
| 처리 | 단계 시작, 반복 요청, 예외 피드백 시 음성으로 출력한다. |
| 출력 | 기기/플랫폼 TTS 음성 |
| 수락 기준 | 단계 시작과 타이머 종료 후 다음 단계 안내가 음성으로 재생된다. |

## F-06. STT 음성 명령

| 항목 | 명세 |
|---|---|
| 목적 | 조리 화면에서 사용자의 발화를 텍스트로 변환한다. |
| 입력 | 조리 세션 중 사용자 음성 |
| 처리 | 조리 화면 활성 상태에서만 STT를 실행한다. |
| 출력 | transcript text |
| 예외 | 인식 실패 시 다시 말해달라는 상태와 버튼 fallback 제공 |
| 수락 기준 | "다음", "다시 말해줘", "1분 더", "물이 안 끓어" 같은 발화를 텍스트로 받을 수 있다. |

## F-07. Command Router

| 항목 | 명세 |
|---|---|
| 목적 | 발화를 로컬 명령과 LLM 예외 질문으로 분리한다. |
| 로컬 명령 | 다음, 이전, 반복, 현재 단계 확인, 타이머 연장/정지/재개 |
| LLM 발화 | 물이 안 끓음, 너무 짬, 덜 익음, 재료 없음, 안전 질문 |
| 출력 | local action 또는 LLM request |
| 수락 기준 | 명확한 P0 명령은 LLM 호출 없이 즉시 처리된다. |

## F-08. LLM 예외 피드백

| 항목 | 명세 |
|---|---|
| 목적 | 현재 단계 맥락이 필요한 조리 문제에 짧은 행동 지침을 준다. |
| 입력 | 레시피명, 현재 단계, 타이머, 사용자 발화, 최근 이벤트 |
| 처리 | 1~2문장과 하나의 다음 행동으로 응답한다. |
| 출력 | 화면 텍스트, TTS 응답, 저장할 이벤트 |
| 수락 기준 | "물이 안 끓어"에 대해 현재 단계 기준의 연장/대기 안내를 제공한다. |

## F-09. 버튼 fallback

| 항목 | 명세 |
|---|---|
| 목적 | 소음, 마이크 권한, STT 실패 상황에서도 조리를 끝낼 수 있게 한다. |
| 버튼 | 이전, 다시 듣기, 1분 추가, 다음, 완료 |
| 처리 | 버튼 이벤트를 음성 명령과 동일한 이벤트 타입으로 저장한다. |
| 수락 기준 | 음성 없이도 레시피를 처음부터 끝까지 진행할 수 있다. |

## F-10. 조리 후 피드백

| 항목 | 명세 |
|---|---|
| 목적 | 조리 결과를 개인 레시피 개선 데이터로 저장한다. |
| 입력 | 만족도, 자유 코멘트, 다음에 바꿀 점 |
| 처리 | 짰다/싱거웠다/덜 익었다/퍼졌다 등 피드백을 구조화한다. |
| 출력 | `post_cook_reviews`, recipe adjustment draft |
| 수락 기준 | "너무 짰다"가 다음 조리 수정안으로 변환된다. |

## F-11. 개인 레시피 버전 생성

| 항목 | 명세 |
|---|---|
| 목적 | 같은 레시피를 더 잘 만들 수 있는 사용자별 버전을 저장한다. |
| 입력 | 조리 후 피드백(프론트가 넘긴 `recipeId` + 평가/코멘트) |
| 처리 | 기본 레시피에 시간/재료/간 조정값을 반영한다. |
| 출력 | `personal_recipe_versions` |
| 수락 기준 | 조리 후 피드백을 반영한 개인 버전이 생성된다. |

## F-12. 다음 조리 개인화

| 항목 | 명세 |
|---|---|
| 목적 | 같은 레시피 재진입 시 개선된 내 버전을 제공한다. |
| 입력 | 레시피 ID, 사용자 ID |
| 처리 | 최신/만족도 높은 개인 버전을 조회한다. |
| 출력 | 개인화된 단계, 시간, 재료 조정값 |
| 수락 기준 | 같은 레시피를 다시 선택하면 이전 피드백이 반영된 버전이 보인다. |
